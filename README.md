# Project Board — доска проектов

Простое веб-приложение для управления проектами в стиле Kanban. Проекты можно создавать, перемещать между колонками и удалять. Все данные сохраняются в браузере автоматически.

---

## Что нужно установить перед запуском

1. **Node.js** — среда для запуска JavaScript на компьютере.  
   Скачать: https://nodejs.org/ (выберите версию LTS).

2. **Терминал** — командная строка:
   - Windows: PowerShell или «Терминал» в VS Code / Cursor
   - macOS / Linux: Terminal

Проверьте, что Node.js установлен:

```bash
node --version
npm --version
```

Должны появиться номера версий (например, `v22.x.x` и `10.x.x`).

---

## Как запустить проект

### 1. Откройте папку проекта в терминале

```bash
cd путь/к/to_do_list
```

Например, если проект лежит в `H:\todo_lebedev\Web\to_do_list`:

```bash
cd H:\todo_lebedev\Web\to_do_list
```

### 2. Установите зависимости (только при первом запуске)

```bash
npm install
```

Команда скачает все нужные библиотеки в папку `node_modules`. Это может занять 1–2 минуты.

### 3. Запустите сервер разработки

```bash
npm run dev
```

или то же самое:

```bash
npm start
```

В терминале появится адрес, обычно:

```
http://127.0.0.1:5174
```

Откройте этот адрес в браузере (Chrome, Firefox, Edge и т.д.).

Чтобы остановить сервер, нажмите `Ctrl + C` в терминале.

---

## Как пользоваться приложением

1. Введите **название проекта** в верхнее поле.
2. При желании добавьте **описание** во второе поле.
3. Нажмите **«Создать»** — проект появится в колонке **«Новое»**.
4. Кнопка **«Далее»** на карточке перемещает проект:  
   Новое → В процессе → Готово → снова Новое.
5. Кнопка **«Удалить»** убирает проект навсегда.

Данные хранятся в `localStorage` браузера — после перезагрузки страницы проекты останутся на месте.

## Структура проекта

```
to_do_list/
├── index.html          # Главная HTML-страница (точка входа в браузер)
├── package.json        # Список зависимостей и команды npm
├── vite.config.js      # Настройки сборщика Vite (порт, плагины)
├── eslint.config.js    # Правила проверки кода (линтер)
│
├── public/             # Статические файлы (иконки, favicon)
│   ├── favicon.svg
│   └── icons.svg
│
└── src/                # Исходный код приложения
    ├── main.jsx        # Точка входа React — подключает App к странице
    ├── App.jsx         # Главный компонент: состояние, колонки, localStorage
    ├── AddTask.jsx     # Форма добавления нового проекта
    ├── Task.jsx        # Карточка одного проекта (кнопки «Далее» и «Удалить»)
    ├── App.css         # Стили интерфейса
    ├── index.css       # Базовые глобальные стили
    └── assets/         # Картинки (логотипы React и Vite)
```

### Кратко о файлах

| Файл | За что отвечает |
|------|-----------------|
| `main.jsx` | Запускает React и вставляет приложение в `<div id="root">` |
| `App.jsx` | Хранит список проектов, считает статистику, рисует три колонки |
| `AddTask.jsx` | Поля ввода и кнопка «Создать» |
| `Task.jsx` | Одна карточка проекта с названием, описанием и действиями |
| `App.css` | Внешний вид: колонки, карточки, кнопки |

### Технологии

- **React** — библиотека для построения интерфейса из компонентов
- **Vite** — быстрый инструмент для разработки и сборки
- **localStorage** — встроенное хранилище браузера (без сервера и базы данных)

## Полезные команды

| Команда | Что делает |
|---------|------------|
| `npm run dev` | Запуск в режиме разработки (с автообновлением при изменении кода) |
| `npm start` | То же, что `npm run dev` |
| `npm run build` | Собрать готовую версию для публикации в папку `dist/` |
| `npm run preview` | Посмотреть собранную версию локально |
| `npm run lint` | Проверить код на ошибки и стиль |

### Сборка для продакшна

```bash
npm run build
npm run preview
```

После `build` в папке `dist/` появятся готовые файлы, которые можно выложить на любой хостинг статики.

## Частые проблемы

**«npm не является внутренней командой»**  
Node.js не установлен или не добавлен в PATH. Переустановите Node.js с официального сайта.

**Порт занят**  
Vite попробует другой порт автоматически. Смотрите актуальный адрес в выводе терминала.

**Пустая страница**  
Убедитесь, что сервер запущен (`npm run dev`) и вы открываете именно тот адрес, который показан в терминале.

**Проекты пропали**  
Данные привязаны к браузеру и домену. Очистка cookies/localStorage или другой браузер — пустая доска.
